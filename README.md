# **SpectraAnalysis User Manual**

## **1\. Overview**

SpectraAnalysis is a software tool developed to assist in the visualization, processing, and interpretation of fluorescence emission data generated by spectrofluorometer platforms, particularly SpectraMax-based tabular datasets. The software provides an interactive graphical interface that allows users to upload raw fluorescence files, select positive and negative control wells, classify experimental samples, and generate graphical outputs for spectral analysis.

The program was initially designed to support the interpretation of fluorescence-based diagnostic assays, such as FRET-HCR approaches for H5N1 detection. However, it can also be applied to other fluorescence-based experiments that generate wavelength-dependent emission data in a plate-based format.

## **2\. Input File Requirements**

SpectraAnalysis is designed to process tabular files exported from SpectraMax instruments. The input file must contain fluorescence emission measurements organized by wavelength and well position.

The expected file structure includes:

* A wavelength column named `Wavelength`;  
* Sample columns identified by 96-well plate positions, such as `A1`, `A2`, `B1`, `H12`;  
* Fluorescence intensity values for each wavelength;  
* Tab-separated values;  
* UTF-16 encoding;  
* Decimal values using comma notation, when applicable.

The software automatically ignores the temperature column if it is present in the file. Completely empty wells are also identified and excluded from the analysis.

## **3\. Starting the Program**

After opening SpectraAnalysis, the initial page displays a short description of the software and the data upload interface. To begin the analysis, click the **Upload File** button and select the SpectraMax output file from your computer.

Once the file is uploaded, the software reads the fluorescence dataset and generates an interactive 96-well plate map. Wells containing valid fluorescence data are displayed as active buttons, while empty wells are disabled.

## **4\. Selecting Positive and Negative Controls**

After uploading the file, users must define the positive and negative control wells directly on the 96-well plate map.

Each active well button changes its state according to the number of clicks:

* **First click:** the well is assigned as a negative control and turns blue;  
* **Second click:** the well is assigned as a positive control and turns red;  
* **Third click:** the well returns to the unassigned state.

Users should select at least one negative control and one positive control before generating the results. The negative controls are used to calculate the baseline signal and classification threshold, while the positive controls provide a reference for expected positive fluorescence behavior.

After selecting the controls, click **Generate Results** to proceed.

## **5\. Control Validation**

Before classifying the samples, SpectraAnalysis checks whether the selected controls are consistent with one another. For each control group, the software evaluates the fluorescence peak within the selected emission interval. If the difference between controls in the same group is too large, the software displays an error message indicating that the controls may have been poorly selected.

This step helps prevent unreliable classifications caused by inconsistent or incorrectly assigned controls.

## **6\. Sample Classification**

After the controls are validated, SpectraAnalysis calculates the mean fluorescence signal of the negative controls and uses it as the baseline for sample classification.

For each sample well, the software normalizes the fluorescence signal relative to the negative control mean and evaluates the emission peak within the predefined wavelength interval. In the current version, the peak is calculated from the strongest fluorescence values between 655 and 665 nm.

Samples with normalized peak values above the calculated threshold are classified as positive. Samples with normalized peak values below the threshold are classified as negative. Empty wells are identified and excluded from the classification process.

## **7\. Results Page**

After classification, the software opens the results page and displays a new 96-well plate map. Each well is colored according to its classification or control status.

The color scheme is:

* **Dark red:** positive control;  
* **Dark blue:** negative control;  
* **Light red:** positive sample;  
* **Light blue:** negative sample;  
* **Grey or disabled wells:** empty or unavailable wells.

This visual layout allows users to rapidly inspect the spatial distribution of positive and negative samples across the plate.

## **8\. Selecting Wells for Graph Visualization**

On the results page, users can select wells to be included in the graphical output. Wells can be selected individually or collectively.

Available selection options include:

* Clicking an individual well to select or deselect it;  
* Clicking a row label, such as `A`, `B`, or `C`, to select an entire row;  
* Clicking a column number, such as `1`, `2`, or `12`, to select an entire column;  
* Clicking **ALL** to select all valid wells simultaneously.

Selected wells change to a darker color with white lettering, making it easier to identify which samples will be plotted.

After selecting the desired wells, click **Plot Cells** to generate the graphical output.

## **9\. Graphical Outputs**

SpectraAnalysis generates graphical representations of the selected wells to facilitate interpretation of the fluorescence data.

The first graph is a line plot showing the fluorescence emission curves. In this plot, the x-axis represents the emission wavelength in nanometers, and the y-axis represents fluorescence intensity in arbitrary units. Sample curves are colored according to their classification as positive or negative. The mean positive and negative control curves are also displayed when selected, and their standard deviation is represented as a shaded region around the corresponding curve.

The graph also highlights the wavelength interval used for peak calculation, allowing users to visually inspect the region that contributes to sample classification.

The second graph is a bar plot showing the normalized fluorescence peak values for the selected samples and controls. The y-axis represents the normalized peak or fold-change relative to the negative control baseline, while the x-axis represents the selected well names. The classification threshold is displayed as a dashed horizontal line, allowing direct comparison between each sample and the positivity threshold.

## **10\. Exporting Graphs**

When the graphical output is generated, SpectraAnalysis saves the figure as a PNG image file in the program directory. The file name includes a timestamp, which prevents previous plots from being overwritten during the same session.

These PNG files can be used for documentation, reports, supplementary figures, or further analysis.

## **11\. Restarting the Analysis**

To start a new analysis, click the **Back** button on the results page. This resets the interface and returns the user to the initial page, where a new file can be uploaded and analyzed.

## **12\. Error Messages and Troubleshooting**

### **“Select at least one cell to generate the graph.”**

This message appears when the user clicks **Plot Cells** without selecting any wells on the results page. To solve this, select at least one well, row, column, or use the **ALL** button before generating the plot.

### **“There are no valid negative controls to calculate the threshold.”**

This message appears when no valid negative control is available for threshold calculation. To solve this, return to the control selection step and make sure at least one valid negative control well is selected.

### **“There are no data points between 655 and 665 nm.”**

This message appears when the uploaded dataset does not contain wavelength values within the interval used for peak calculation. To solve this, check whether the input file contains the expected emission wavelength range.

### **“It was not possible to calculate the peaks of the negative controls.”**

This message appears when the software cannot extract valid peak values from the selected negative controls. This may occur if the selected wells are empty, contain missing values, or do not have valid fluorescence data in the required wavelength interval.

### **“The controls were poorly selected.”**

This message appears when the selected controls within the same group show excessive variation. To solve this, verify whether the correct wells were assigned as positive or negative controls and repeat the control selection step.

## **13\. Recommended Workflow**

1. Open SpectraAnalysis.  
2. Click **Upload File**.  
3. Select the SpectraMax fluorescence data file.  
4. Inspect the generated 96-well plate map.  
5. Select the negative control wells by clicking once on each corresponding well.  
6. Select the positive control wells by clicking twice on each corresponding well.  
7. Click **Generate Results**.  
8. Inspect the classified plate map.  
9. Select individual wells, rows, columns, or all wells for plotting.  
10. Click **Plot Cells**.  
11. Review the line plot and normalized peak bar plot.  
12. Use the generated PNG file for documentation or further analysis.

## **14\. Notes and Limitations**

SpectraAnalysis depends on correct control selection. Incorrectly assigned positive or negative controls may affect the classification threshold and lead to inaccurate sample interpretation.

The current version calculates peak-based classification using a predefined wavelength interval between 655 and 665 nm. Therefore, datasets that do not include this emission range may not be compatible with the default analysis settings.

The software is optimized for fluorescence datasets organized according to a 96-well plate format. Files with different column structures may require formatting before being uploaded.
